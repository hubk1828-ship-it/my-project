# رؤية المشروع — منصة تحليل وتداول العملات الرقمية

---

## 1. وصف المشروع والمشكلة

### المشكلة
- أغلب منصات التحليل تعتمد على شائعات وتوصيات مضللة من وسائل التواصل الاجتماعي
- لا تُفصح عن غياب الفرصة، بل تختلق توصيات وهمية لإبقاء المستخدم متفاعلاً
- لا ربط حقيقي بين الأخبار الموثوقة وحركة السعر الفعلية

### الحل
منصة ويب داخلية تعمل كأداة تحليل واقعية ومحايدة:
- تعتمد **حصرياً** على بيانات السوق اللحظية والمصادر الموثوقة
- تتجاهل كلياً الشائعات ووسائل التواصل الاجتماعي
- تُفصح صراحةً عندما **لا توجد فرصة واضحة** بدلاً من اختراع توصيات
- تربط الأخبار المؤثرة بحركة الأسعار الحقيقية بقرارات تحليلية صادقة

## 2. الميزات الأساسية (MVP)

### 2.1 نظام المستخدمين (مغلق)
| الخاصية | التفاصيل |
|---|---|
| التسجيل | بدعوة من الأدمن فقط (لا تسجيل مفتوح) |
| الأدوار | `admin` · `user` |
| المصادقة | JWT مع refresh tokens |
| صلاحيات الأدمن | إضافة/حذف/تعليق مستخدمين، التحكم بحدود التداول الآلي |

### 2.2 محرك التحليل
| الخاصية | التفاصيل |
|---|---|
| مصادر البيانات | Binance API + Bybit API (بيانات لحظية عبر WebSocket) |
| المخرجات الثلاثية | `شراء` · `بيع` · `لا توجد فرصة واضحة` |
| منهجية التحليل | تحليل فني (مؤشرات) + ربط بالأخبار الموثوقة |
| الشفافية | عرض أسباب القرار ومستوى الثقة لكل تحليل |
| التحديث | لحظي عبر WebSocket |

### 2.3 المحافظ الرقمية
| الخاصية | التفاصيل |
|---|---|
| الربط | محفظة منفصلة لكل مستخدم عبر API keys |
| المنصات | Binance · Bybit |
| العرض | رصيد المحفظة، الصفقات المفتوحة، تاريخ التداول |
| الأمان | تشفير API keys في قاعدة البيانات |

### 2.4 التداول الآلي
| الخاصية | التفاصيل |
|---|---|
| التفعيل | اختياري — يفعّله المستخدم ويوافق عليه الأدمن |
| الحدود | حد أقصى للصفقة، حد يومي، حد خسارة — يتحكم فيها الأدمن |
| الإيقاف | إيقاف فوري من المستخدم أو الأدمن في أي وقت |
| السجل | تسجيل كل عملية تداول آلي مع السبب |

### 2.5 نظام التنبيهات
| القناة | الاستخدام |
|---|---|
| Telegram Bot | تنبيهات فورية (فرص، تنفيذ صفقات، تحذيرات) |
| Email (SMTP) | ملخصات يومية، تنبيهات أمنية |
| الموقع | إشعارات لحظية داخل المنصة |

## 3. نموذج العمل والفئة المستهدفة

| البند | التفاصيل |
|---|---|
| النوع | أداة داخلية للفريق — ليست منتجاً تجارياً |
| نموذج الربح | لا يوجد — الربح من التداول نفسه وليس من المنصة |
| الفئة المستهدفة | أعضاء الفريق فقط (عدد محدود) |
| إدارة الوصول | الأدمن هو الوحيد الذي يضيف المستخدمين |
| قابلية التوسع | مصممة لفريق صغير (5-20 مستخدم) |

## 4. التقنيات النهائية

### Stack كامل

```
┌─────────────────────────────────────────┐
│              Frontend                    │
│         Next.js (React/TypeScript)       │
│         WebSocket Client                 │
├─────────────────────────────────────────┤
│              Backend API                 │
│         Python FastAPI                   │
│         WebSocket Server                 │
├─────────────────────────────────────────┤
│            Services Layer                │
│  Binance API · Bybit API · Analysis      │
│  python-telegram-bot · SMTP              │
├─────────────────────────────────────────┤
│             Database                     │
│           PostgreSQL                     │
├─────────────────────────────────────────┤
│          Infrastructure                  │
│    Hetzner VPS · GitHub · CI/CD          │
└─────────────────────────────────────────┘
```

### تفصيل التقنيات

| الطبقة | التقنية | السبب |
|---|---|---|
| الواجهة | **Next.js** | SSR, routing مدمج, أداء عالي |
| الخلفية | **Python FastAPI** | async أصلي, سرعة عالية, WebSocket مدمج |
| قاعدة البيانات | **PostgreSQL** | موثوقة, تدعم JSON, مناسبة للبيانات المالية |
| البيانات اللحظية | **WebSocket** | اتصال مستمر لبيانات الأسعار اللحظية |
| تداول | **Binance API + Bybit API** | التنفيذ المباشر والبيانات اللحظية |
| تنبيهات | **python-telegram-bot + SMTP** | Telegram للفوري, Email للملخصات |
| الاستضافة | **Hetzner VPS** | أداء/سعر ممتاز, سيرفرات أوروبية |
| الكود | **GitHub** | إدارة الكود + CI/CD |
